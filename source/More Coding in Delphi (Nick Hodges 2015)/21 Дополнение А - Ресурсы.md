## Системы контроля версий (Source Control)

Большинство фреймворков, которые я рекомендую в книге, получаются через публичные репозитории систем контроля версий. Вот несколько ссылок, чтобы начать, если вы не знакомы с Git, Mercurial или Subversion.

### Subversion

Subversion — это популярная, серверная система контроля версий, пришедшая на смену CVS. Серверная система контроля версий означает, что репозиторий хранится централизованно на сервере, и клиенты отправляют запросы на чтение и запись в этот центральный репозиторий. Subversion была популярна в течение многих лет и остается таковой. Однако я рекомендую, чтобы вы использовали одну из распределенных систем контроля версий (VCS), таких как Git или Mercurial, когда это возможно.

В любом случае, вот подробности для Subversion:

*   Скачать Subversion можно здесь: https://subversion.apache.org/
*   Документацию можно найти здесь: https://subversion.apache.org/docs/

### Git

Git — это распределенная система контроля версий, изначально написанная Линусом Торвальдсом для поддержки разработки Linux.

*   Git можно скачать здесь: http://git-scm.com/downloads
*   Хороший, пошаговый учебник, обучающий основам Git: http://gitimmersion.com/
*   Отличная онлайн-книга, обучающая Git: http://git-scm.com/book/en/

### Mercurial

Mercurial (hg) — это еще одна популярная распределенная система контроля версий.

*   Mercurial можно скачать здесь: http://mercurial.selenic.com/wiki/Download
*   Простой и хорошо сделанный учебник для Mercurial — это hginit.com. Он специально разработан для людей, переходящих к распределенной системе контроля версий из мира серверных систем.
*   Команда Mercurial имеет хороший учебник: http://mercurial.selenic.com/wiki/Tutorial
